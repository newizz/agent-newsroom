Concept Explainer

How Large Language Models Work

An LLM is, at heart, a very large statistical next-token predictor. The "magic" is six mechanisms — tokens, embeddings, attention, training, sampling, and context — combined at massive scale.

Overview How it works Examples Glossary Sources

Overview

Before 2017, language models were RNNs and LSTMs — networks that processed text left-to-right with a hidden state that struggled to remember things from far back in the sequence. The Transformer architecture[1] threw all that out and replaced it with self-attention, which lets every word interact with every other word in parallel.

That parallelism is what made today's models possible. RNNs couldn't be efficiently trained on huge datasets; Transformers can. Scaled-up Transformers became GPT, BERT, Claude, Gemini, and every modern LLM.

Underneath all the polish, an LLM does one thing: given some text, predict the most likely next chunk of text. Everything you experience — chat, summarisation, code generation, reasoning — emerges from that single trick applied billions of times across trillions of training examples.

How it works

Tokenize — chop text into pieces

A tokenizer breaks your prompt into integer IDs from a fixed vocabulary of ~100,000 tokens. Common words become one token; rare words split into pieces. Rough rule: 1 token ≈ 4 characters in English[4].

Embed — turn IDs into meaning vectors

Each token ID is looked up in a giant table and replaced with a vector of ~4,000–12,000 numbers. Similar meanings end up close together in this space. Famously: king − man + woman ≈ queen[2].

Attend — every token looks at every other

For each token, the model computes a Query, Key, and Value. The Query is matched against every other token's Key to decide how much attention to pay — then those attention weights mix the Values to update the representation[1].

Stack — repeat this dozens of times

Frontier models stack 80–120 layers of attention + feed-forward blocks. Lower layers learn grammar; higher layers learn facts, intent, and style — without anyone explicitly programming those rules[3].

Predict — produce a probability for every next token

The final layer projects to a 100,000-wide vector — one logit per vocabulary token. Softmax turns those logits into a probability distribution. The model now "thinks" every possible next word with a probability.

Sample — pick one token, then loop

A sampler picks one token from the distribution (greedy, temperature, or top-p). The new token is appended and the whole process runs again to produce the next one. Token by token, sentences and pages emerge.

Worked examples

EXAMPLE 1

Tokenizing "extraordinary"

A rare word like extraordinary doesn't get its own token. The BPE tokenizer splits it into common pieces:

"extraordinary" → ["extra", "ordinary"]
→ [11814, 19929]

Two tokens for one "word" — and the model sees the link to other "ordinary" contexts it has learned.

EXAMPLE 2

Predicting "The cat sat on the ___"

After tokenization, embedding, and 100 layers of attention, the final probabilities for the next token look something like:

mat
41%
floor
18%
chair
9%
couch
6%
... ~99,996 more

Greedy sampling picks mat. Higher temperature might pick floor instead.

Key terms

Token
A sub-word piece of text; the atomic unit the model operates on. Roughly 4 characters of English.
Embedding
A learned vector representation of a token. Similar meanings → similar vectors.
Attention
The mechanism that lets each token "look at" every other token to update its representation based on context.
Parameter
A single trainable number in the model. Frontier models have 10¹¹–10¹² of them (100B–1T).
Inference
Running the trained model to produce output. Contrast with training, which updates the parameters.
Temperature
A knob (0 to ~2) that controls how random sampling is. 0 = always pick the most likely token; higher = more creative / variable.
Context window
The maximum number of tokens the model can attend to at once. Costs scale as O(n²), which is why long context is engineering-intensive.
